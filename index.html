How to Contribute

This knowledge base lives and grows with your input. Here's how to add or update content.

1

Identify What's Missing or Outdated

Think about questions you've had to ask multiple times, docs that helped you, or concepts that clicked after talking to someone. That's usually a sign something should be documented here.

2

Open the Content File

All content lives in a single JSON file that anyone on the team can edit. No coding skills needed — just follow the structure.

Location: knowledge-base-data.json (or ask Josch/Daniel for access)

3

Add Your Entry

Copy an existing entry and modify it. Here's the template:

{ "id": 7, "title": "Your Concept Title", "category": "orchestration", "summary": "One sentence description", "author": "Your Name", "date": "May 2025", "lastUpdated": "2025-05-15", "references": [ { "title": "Link to source doc", "url": "https://..." } ], "content": "Your full explanation..." }

Categories: orchestration, agents, memory, rag, evals, architecture

4

Write Like a Designer, for Designers

Use Neslihan's "What is Orchestration" as a template:

  • Start with "What is it?" — plain language definition
  • Add an analogy or metaphor in a callout box
  • "Why does it matter for designers?" — connect to our work
  • "Key things to know" — bullets with practical takeaways
  • Link to source materials in the references section
5

Add References

Always link back to source material — internal docs, Slack threads, Johannes' drafts, architecture diagrams. This builds trust and lets people go deeper.

6

Save & Share

Once you've added your entry, save the file and let the team know in Slack. Your contribution will appear on the site automatically.

Questions?

Reach out to Josch or Daniel — we're here to help you get your first entry published.
